# ADR-001 — Escolha da Biblioteca de Acesso ao Banco de Dados

## Status

Aprovado

---

## Contexto

Foi realizada uma Prova de Conceito (PoC) em Python para comparar o desempenho de duas bibliotecas de acesso ao banco de dados SQL Server:

* `pyodbc`
* `SQLAlchemy`

O objetivo foi medir o tempo de execução de uma consulta SQL e o tempo para carregar os dados em memória.

A mesma consulta foi utilizada em ambas as bibliotecas:

```sql id="sql001"
SELECT TOP 5 * FROM erp.cliente
```

Cada teste foi executado duas vezes para cálculo do tempo médio.

---

## Decisão

A biblioteca escolhida foi o **pyodbc**.

A decisão foi tomada porque o `pyodbc` apresentou melhor desempenho durante os testes da PoC.

---

## Opções Consideradas

### PyODBC

* Driver nativo para conexão com SQL Server
* Maior desempenho nos testes
* Menor overhead

### SQLAlchemy

* Biblioteca ORM e camada de abstração
* Código mais organizado e flexível
* Tempo de execução maior no cenário testado

---

## Consequências

### Positivas

* Melhor tempo de resposta
* Menor tempo de execução das consultas
* Implementação simples e direta

### Negativas

* Menor nível de abstração
* Mais código manual para consultas SQL
